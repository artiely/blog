# 写 vue 最佳编辑器 vscode

Vue.js 是一个用于构建 Web 应用程序用户界面的流行 JavaScript 库，Visual Studio Code 内置了对 HTML，CSS 和 JavaScript 的 Vue.js 构建块的支持。对于更丰富的 Vue.js 开发环境，您可以安装支持 Vue.js IntelliSense，代码片段，格式等的Vetur扩展。

# 我们要学什么？

  • 在我们的.vue 文件中获取语法高亮
  • 利用代码片段实现更快的工作流程
  • 配置我们的编辑器以自动格式化我们的代码
  • 并探索其他有用的扩展，这些扩展将改善我们的开发体验

# Vetur 扩展

它是一个功能丰富的扩展，为您提供诸如.vue 文件中的语法突出显示，片段，错误检查和格式化以及自动完成和调试等功能。在这一点上，它是代码编辑器的最佳 Vue 扩展。它应该是，因为它是由 Pine Wu 开发的，他是 Vue 核心团队的成员。

通过 Vue 扩展语言服务的 VS Code 还可以通过 Go to Definition（F12）或 Peek Definition（Alt + F12）在编辑器中提供类型定义信息。按 Escape 键关闭 Peek 窗口。

# Vetur 的特点

现在已经安装了 Vetur，让我们来看看它的功能。

语法突出显示通过键入 command + P 并键入.vue 文件的名称，我们可以打开 About.vue 文件。如您所见，现在我们的代码正在获得正确的语法突出显示。太棒了 - 没有更多灰色代码。

检查 Home.vue 文件，我们可以看到我们的 JavaScript 也正确地突出显示。

片段另一个功能 Vetur 与他的代码片段打包在一起。这些是节省时间的“代码片段”，允许您快速创建常用的代码块。

让我们创建一个新组件来实现这一点。我们的名字是 EventCard.vue。现在，如果我们在.vue 文件中键入单词“scaffold”并按 Enter 键，这将使用单个文件.vue 组件的骨架或脚手架自动填充该文件。

Emmet Vetur 也与 Emmet 一起使用。这是一个流行的工具，允许您使用快捷方式来构建代码。

要了解有关 Emmet 如何加速您的开发的更多信息，请访问此处。

# 安装 ESLint 和 Prettier

现在，我们需要确保安装了 ESLint 和 Prettier。在扩展存储中，我们将搜索 ESLint，然后继续安装它。我们将为 Prettier 做同样的事情。一旦安装完毕，我们就会重新加载以重新加载 VS Code。

# 配置 ESLint

现在已经安装了这些，我们需要为它们添加一些额外的配置。

当我们从终端创建我们的项目时，我们选择使用专用的配置文件创建它，它给了我们这个.eslintrc.js文件，我们可以为这个项目配置 ESLint。如果我们没有选择专用文件，我们会在package.json中找到 ESLint 配置。

因此，在我们的.eslintrc.js文件中，我们将添加：

'plugin:prettier/recommended'
1

这将使用默认设置在 ESLint 中启用 Prettier 支持。

所以我们的文件现在看起来像这样：

module.exports = {
  root: true,
  env: {
    node: true
  },
  extends: [
    'plugin:vue/essential',
    'plugin:prettier/recommended', // we added this line
    '@vue/prettier'
  ],
  rules: {
    'no-console': process.env.NODE_ENV === 'production' ? 'error' : 'off',
    'no-debugger': process.env.NODE_ENV === 'production' ? 'error' : 'off'
  },
  parserOptions: {
    parser: 'babel-eslint'
  }
}

# 配置 prettier

我们还可以选择创建一个 Prettier 配置文件，根据我们的个人风格或团队的偏好添加一些特殊设置。

我们将在这里创建并命名 .prettierrc.js。

在里面，我们将输入：

module.exports = {
  singleQuote: true,
  semi: false
}

# 用户设置

为了进一步优化 VS Code 以获得出色的开发体验，我们将为用户设置添加一些配置。要访问“用户设置”，请单击顶部导航栏中的“代码 ”，然后单击“ 首选项”，再单击“ 设置”。这将打开一个用户设置窗口，您可以在其中添加 json 中的设置。

首先，我们要添加：

"vetur.validation.template": false
1

这将关闭 Vetur 的 linting 功能。我们将依赖于 ESLint + Prettier。

现在我们想告诉 ESLint 我们希望它验证哪些语言（vue，html 和 javascript）并设置 autoFix 为 true 每个语言：

"eslint.validate": [
    {
        "language": "vue",
        "autoFix": true
    },
    {
        "language": "html",
        "autoFix": true
    },
    {
        "language": "javascript",
        "autoFix": true
    }
],

现在让我们来看看一些可以帮助加快开发的其他工具。

# Copy Relative Path

Copy Relative Path(复制相对路径)扩展，它允许您根据文件与其链接目录的关系来复制文件所在的位置。

让我们搜索它，安装它，然后看怎么使用。

在我们的 Home.vue 文件中，我们已经看到这里有一个相对路径，我们正在导入 HelloWorld 组件。

为了获取我们要导入的文件的相对路径，我们右键单击该文件，然后选择 Copy Relative Path。现在，当我们粘贴复制的内容时，我们看到我们有准确的相对路径。

# 集成终端

VScode 编辑器的一个方便的内置功能是它的集成终端，您可以使用它而不是切换到您的单独终端。您可以使用键盘快捷键打开它：ctrl +`

更多代码片段如果您有兴趣安装一些其他方便的代码片段，可以下载由 Vue Team 成员 Sarah Drasner 创建的全套 Vue VSCode 代码片段。

让我们用她的名字 sarah.drasner 搜索扩展名。他们在那里。现在我们可以安装和重新加载。

我们来看看它们的实际效果。

如果我们vif在模板中输入一个元素，那就会给我们一个v-if语句，输入von将给我们一个完整的事件处理程序。我们可以简单地键入vdata哪个将为我们创建一个数据属性，而不是手动输入数据属性。我们可以做同样的事情来添加道具vprops。我们甚至可以使用它来创建快速导入库的代码vimport-lib。如您所见，这些是非常有用且节省时间的片段。

请注意，如果您使用此代码段扩展名，建议您在“用户设置”中添加一行：

vetur.completion.useScaffoldSnippets : false
1

这将确保这些片段与 Vetur 没有冲突。

如你所见，我正在使用 FlatUI Dark。您可以在此处将主题颜色更改为任何这些选项，也可以在扩展存储中搜索其他主题。
